# Care Passport — Question Framework

## Question Design Principles

Questions should be broadly applicable across multiple support settings.

Avoid assumptions that the user:

- lives in residential aged care
- has a bedside table
- receives overnight support
- has a particular diagnosis
- has age-related needs

Where examples are required, examples should be supportive rather than prescriptive.

The MVP will be validated primarily in residential aged care, but the core question framework should remain suitable for broader support contexts.

## Question Pathways

The MVP supports three completion pathways:

- Essential 12
- Recommended 25
- Full 37

No question is required. Users may skip any question.

## Context-Specific Question Sets

The MVP will use a single universal question framework.

Future versions may introduce optional question packs for:

- residential aged care
- disability support
- home care
- hospital admissions
- rehabilitation
- palliative care
- respite care

The core identity questions should remain shared across all contexts.

## Visibility

Each answer must be assigned one of the following visibility states:

- Poster
- Booklet
- Private family record / exclude from printed outputs

Sensitive answers must be able to be excluded from all printed outputs.

## Core Question Set

The supplied 37-question set remains the Full pathway for the MVP.

The Essential 12 and Recommended 25 subsets must be stored in the database as pathway mappings, not hardcoded.

## Question Content Management

Questions, sections, help text, labels, ordering, pathway membership and poster mappings must be database-driven.

A future admin interface may allow non-technical editing of question wording and mappings.
